Git — это распределённая система контроля версий (Version Control System, VCS), предназначенная для отслеживания изменений в файлах (чаще всего — исходном коде) и координации работы нескольких людей над одним проектом.

Основные задачи, для которых нужен Git:
Хранение истории изменений
Фиксирует все изменения в файлах с возможностью отката к любой предыдущей версии.

Координация командной работы
Позволяет нескольким разработчикам работать параллельно над одним проектом без конфликтов.

Ветвление и слияние (Branching/Merging)
Создание изолированных веток для разработки новых функций или исправления ошибок с последующим объединением в основной код.

Резервное копирование и восстановление
Хранилище кода с полной историей на каждом компьютере разработчика (распределённая система).

Отслеживание авторства
Каждое изменение подписывается автором, что упрощает анализ истории и ответственность.

Тестирование и эксперименты
Можно создавать временные ветки для тестирования идей без риска повредить основной проект.
